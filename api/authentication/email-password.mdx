## Sign Up

Create a new user account with email and password.

```bash
POST /api/auth/sign-up/email
```

### Request Body

<ParamField body="email" type="string" required>
  User's email address. Must be valid and unique.
</ParamField>

<ParamField body="password" type="string" required>
  User's password. Should meet minimum security requirements.
</ParamField>

<ParamField body="name" type="string">
  User's display name.
</ParamField>

### Request Example

```bash
curl -X POST "{NEXT_PUBLIC_APP_URL}/api/auth/sign-up/email" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "securePassword123",
    "name": "John Doe"
  }'
```

### Response

<ResponseField name="user" type="object">
  The created user object
</ResponseField>

<ResponseField name="user.id" type="string">
  Unique user identifier
</ResponseField>

<ResponseField name="user.email" type="string">
  User's email address
</ResponseField>

<ResponseField name="user.name" type="string">
  User's display name
</ResponseField>

<ResponseField name="user.email_verified" type="boolean">
  Email verification status (false on signup)
</ResponseField>

<ResponseField name="user.created_at" type="timestamp">
  Account creation timestamp
</ResponseField>

<ResponseField name="session" type="object">
  Created session object
</ResponseField>

### Response Example

```json
{
  "user": {
    "id": "usr_abc123",
    "email": "user@example.com",
    "name": "John Doe",
    "email_verified": false,
    "created_at": "2024-01-15T10:30:00Z",
    "updated_at": "2024-01-15T10:30:00Z"
  },
  "session": {
    "id": "ses_xyz789",
    "user_id": "usr_abc123",
    "expires_at": "2024-01-16T10:30:00Z"
  }
}
```

<Note>
  After signup, a verification email is automatically sent. Users must verify their email before creating organizations.
</Note>

## Sign In

Authenticate with email and password.

```bash
POST /api/auth/sign-in/email
```

### Request Body

<ParamField body="email" type="string" required>
  User's email address
</ParamField>

<ParamField body="password" type="string" required>
  User's password
</ParamField>

### Request Example

```bash
curl -X POST "{NEXT_PUBLIC_APP_URL}/api/auth/sign-in/email" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "securePassword123"
  }'
```

### Response

<ResponseField name="user" type="object">
  The authenticated user object
</ResponseField>

<ResponseField name="session" type="object">
  Created session object with active organization info
</ResponseField>

<ResponseField name="subscription" type="object">
  User's current subscription status
</ResponseField>

<ResponseField name="subscription.isSubscribed" type="boolean">
  Whether user has an active subscription
</ResponseField>

<ResponseField name="subscription.productId" type="string">
  Active product ID (pro, teams, etc.)
</ResponseField>

### Response Example

```json
{
  "user": {
    "id": "usr_abc123",
    "email": "user@example.com",
    "name": "John Doe",
    "email_verified": true,
    "created_at": "2024-01-15T10:30:00Z"
  },
  "session": {
    "id": "ses_xyz789",
    "user_id": "usr_abc123",
    "active_organization_id": "org_456",
    "expires_at": "2024-01-16T10:30:00Z"
  },
  "subscription": {
    "isSubscribed": true,
    "productId": "prod_pro_monthly"
  }
}
```

<Note>
  If the user's email is not verified, a verification email will be sent on sign-in.
</Note>

## Verify Email

Verify user's email address using the token from the verification email.

```bash
GET /api/auth/verify-email
```

### Query Parameters

<ParamField query="token" type="string" required>
  Email verification token from the verification email link
</ParamField>

### Request Example

```bash
curl -X GET "{NEXT_PUBLIC_APP_URL}/api/auth/verify-email?token=verify_token_123"
```

### Response

```json
{
  "success": true,
  "message": "Email verified successfully"
}
```

## Request Password Reset

Request a password reset email.

```bash
POST /api/auth/forget-password
```

### Request Body

<ParamField body="email" type="string" required>
  Email address of the account to reset
</ParamField>

<ParamField body="redirectTo" type="string">
  Optional redirect URL after password reset
</ParamField>

### Request Example

```bash
curl -X POST "{NEXT_PUBLIC_APP_URL}/api/auth/forget-password" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "redirectTo": "/dashboard"
  }'
```

### Response

```json
{
  "success": true,
  "message": "Password reset email sent"
}
```

## Reset Password

Reset password using the token from the reset email.

```bash
POST /api/auth/reset-password
```

### Request Body

<ParamField body="token" type="string" required>
  Password reset token from the email
</ParamField>

<ParamField body="password" type="string" required>
  New password
</ParamField>

### Request Example

```bash
curl -X POST "{NEXT_PUBLIC_APP_URL}/api/auth/reset-password" \
  -H "Content-Type: application/json" \
  -d '{
    "token": "reset_token_123",
    "password": "newSecurePassword456"
  }'
```

### Response

```json
{
  "success": true,
  "message": "Password reset successfully"
}
```

## Generate JWT Token

Generate a JWT token for API access.

```bash
POST /api/auth/token
```

### Request Example

```bash
curl -X POST "{NEXT_PUBLIC_APP_URL}/api/auth/token" \
  -H "Cookie: session=session_token_here"
```

### Response

<ResponseField name="token" type="string">
  JWT token valid for 1 hour
</ResponseField>

### Response Example

```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

<Note>
  JWT tokens expire after 1 hour. Generate a new token before the expiration to maintain API access.
</Note>

## Error Responses

All authentication endpoints may return the following error responses:

### 400 Bad Request

```json
{
  "error": "INVALID_EMAIL",
  "message": "Email address is invalid"
}
```

### 401 Unauthorized

```json
{
  "error": "INVALID_CREDENTIALS",
  "message": "Email or password is incorrect"
}
```

### 409 Conflict

```json
{
  "error": "USER_EXISTS",
  "message": "User with this email already exists"
}
```

### 429 Too Many Requests

```json
{
  "error": "RATE_LIMIT_EXCEEDED",
  "message": "Too many requests, please try again later"
}
```
